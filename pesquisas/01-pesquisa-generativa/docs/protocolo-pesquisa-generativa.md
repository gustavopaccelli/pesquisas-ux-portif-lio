# Protocolo de Pesquisa Generativa
## Comportamento em Redes Sociais: Millennials vs. "Geração Alpha" (Usuários Independentes)

> Documento de trabalho para o Projeto 1 do portfólio de UX Research.
> Métodos: Diary Study + Journey Mapping.
> Status das fontes: dados de mercado citados nas etapas anteriores (TIC Kids Online Brasil 2025, We Are Social/Meltwater Digital 2025-2026, Opinion Box) são reais e verificados via busca; os protocolos abaixo são desenho metodológico original, não uma reprodução de metodologia de terceiros.

---

## 1. Contexto e objetivo de pesquisa

**Pergunta de pesquisa central**: Como o comportamento de consumo em redes sociais difere estruturalmente entre Millennials (~30-44 anos) e adolescentes da "Geração Alpha" (~13-17 anos, tratados aqui como usuários independentes), e que tendências emergentes isso revela?

**Hipóteses a testar/explorar** (não confirmadas — são pontos de partida, não conclusões):
- H1: o motivo de uso difere estruturalmente entre gerações — consumo/decisão de compra rotinizada (Millennials) vs. socialização identitária (Alpha).
- H2: a relação com autonomia e risco percebido no ambiente digital difere entre gerações.
- H3: existe uma mudança de comportamento em curso na geração mais jovem, possivelmente ligada ao debate regulatório recente (ECA Digital) no Brasil.

**Fora de escopo (definido nesta etapa da conversa)**: mediação parental, jornada do responsável, crianças pré-adolescentes (~9-12 anos) com uso dependente de supervisão.

---

## 2. Recorte e critérios de recrutamento

### 2.1 Grupo A — Millennials
| Critério | Definição |
|---|---|
| Idade | 30 a 44 anos |
| Localização | Brasil, preferencialmente diversidade de regiões/classes se possível |
| Critério de inclusão | Usa pelo menos uma rede social (Instagram, WhatsApp, Facebook, TikTok, X) com frequência mínima de uso diário |
| Critério de exclusão | Uso profissional exclusivo (ex: gestor de mídias sociais que só acessa a rede a trabalho) |

### 2.2 Grupo B — "Geração Alpha" (proxy: adolescentes 13-17 anos, usuários independentes)
| Critério | Definição |
|---|---|
| Idade | 13 a 17 anos |
| Autonomia | Autorrelata usar redes sociais "sozinho", sem supervisão direta no momento do uso |
| Critério de inclusão | Uso diário de pelo menos uma plataforma (WhatsApp, Instagram, TikTok) |

**⚠️ Nota ética obrigatória**: mesmo tratando esse grupo como "usuário independente" *no comportamento de uso*, participantes de 13-17 anos são menores de idade no Brasil. Se este projeto envolver recrutamento de participantes reais (e não apenas um estudo simulado/fictício para portfólio), é necessário:
- Termo de Consentimento Livre e Esclarecido (TCLE) assinado por um responsável legal, autorizando a participação na pesquisa.
- Termo de Assentimento assinado pelo próprio adolescente.
- Isso não contradiz o recorte metodológico de "usuário independente" (que se refere ao comportamento de uso da rede social, não ao processo de consentimento da pesquisa) — são duas coisas diferentes e não devem ser confundidas no relatório final.
- Se o projeto for simulado (dados fictícios/sintéticos para fins de portfólio), essa nota ainda deve aparecer no case study como demonstração de rigor metodológico.

**Screener sugerido (perguntas de triagem, ambos os grupos)**:
1. Qual sua idade?
2. Com que frequência você usa redes sociais? (Várias vezes ao dia / uma vez ao dia / algumas vezes por semana / raramente)
3. Cite as 3 redes sociais que você mais usa, em ordem.
4. Nas últimas 24h, você tomou alguma decisão (comprar algo, formar uma opinião, compartilhar algo) motivada por algo que viu em uma rede social? (Sim/Não — usar para priorizar participantes com material rico para o journey mapping)

## 3. Diary Study

### 3.1 Desenho geral
| Parâmetro | Grupo A (Millennials) | Grupo B (Alpha/adolescentes) |
|---|---|---|
| Duração | 10 dias corridos | 7 dias corridos (período mais curto — menor tolerância a tarefas longas) |
| Formato de registro | Formulário simples (Google Forms/Typeform) ou grupo dedicado de WhatsApp | WhatsApp (canal onde já estão, reduz fricção) |
| Frequência de entrada | Ao menos 1 registro por dia, no momento do uso (in-the-moment) | Ao menos 1 registro por dia |
| Incentivo | A definir (vale-presente, sorteio) | A definir — se houver, deve ser combinado com responsável legal |

### 3.2 Kickoff (antes do diário começar)
Sessão curta (15-20 min), individual ou em grupo, para:
- Explicar o propósito da pesquisa em linguagem simples, sem induzir respostas ("queremos entender como você usa redes sociais no dia a dia, não existe resposta certa ou errada").
- Demonstrar como preencher uma entrada de exemplo.
- Combinar horário/canal de lembretes.

### 3.3 Template de entrada diária
Cada entrada deve ser curta (menos de 2 minutos para preencher, para não gerar fadiga e enviesar o comportamento observado):

1. **Que horas são agora?**
2. **Qual rede social você acabou de usar (ou está usando)?**
3. **Por que você abriu o app nesse momento?** (opções rápidas + campo aberto: tédio / notificação / hábito / queria ver algo específico / outro)
4. **O que você fez lá?** (campo aberto curto)
5. **Alguma coisa que você viu te fez pensar em fazer algo depois (comprar, procurar mais informação, comentar, compartilhar)?** (Sim/Não + o quê)
6. **Como você se sentiu depois de usar?** (escala simples: bem / neutro / cansado ou irritado / outro)

### 3.4 Prompt adicional (apenas 1x, no meio do período)
Pergunta de reflexão mais aberta, enviada uma única vez para não sobrecarregar:
- **Millennials**: "Pensando nos últimos dias, teve algum momento em que você usou rede social sem realmente querer/planejar? Como foi isso?"
- **Alpha/adolescentes**: "Teve algum dia em que rede social te ajudou a resolver alguma coisa? E algum dia em que te atrapalhou?"

### 3.5 Debrief (entrevista de encerramento)
Sessão de 20-30 min ao final do período, revisando as entradas do participante junto com ele/ela:
- "Olhando pra tudo que você registrou, o que te surpreende?"
- "Teve algum dia muito diferente dos outros? O que aconteceu?"
- "Se você pudesse mudar uma coisa em como você usa redes sociais, o que seria?"

## 4. Journey Mapping — Roteiro de Entrevista Semiestruturada

**Objetivo da jornada a ser mapeada**: do momento em que a pessoa vê algo em uma rede social até a decisão que toma a partir disso (comprar, formar opinião, compartilhar, agir).

### 4.1 Estrutura da entrevista (45-60 min, individual)

**Abertura (5 min)**
- Apresentação, propósito da entrevista, consentimento de gravação.
- "Antes de começar, me conta um pouco: no seu dia a dia, quais redes sociais você mais usa e para quê?"

**Reconstrução de um episódio recente e concreto (15-20 min)**
Pedir ao participante para pensar em uma vez recente e específica em que algo visto em rede social levou a uma ação/decisão (usar o exemplo do screener, se houver):
- "Me conta sobre essa vez específica — o que você estava fazendo antes de ver aquilo?"
- "O que exatamente você viu? Onde, em qual app, em que formato (story, post, vídeo, anúncio)?"
- "O que passou pela sua cabeça naquele momento?"
- "O que você fez logo em seguida?"
- "Você conversou com alguém sobre isso, ou pesquisou mais em algum outro lugar?"
- "Quanto tempo depois você tomou a decisão final (comprar, comentar, compartilhar, mudar de ideia)?"
- "Olhando pra trás, teve algum momento nessa jornada em que você hesitou, teve dúvida ou ficou desconfortável?"

**Aprofundamento emocional (10 min)**
- "Em que momento dessa jornada você se sentiu mais confiante? E menos confiante?"
- "Se um amigo tivesse passado pela mesma coisa, você teria dado algum conselho diferente do que você mesmo fez?"

**Padrões e generalização (10 min)**
- "Isso que aconteceu é parecido com outras vezes, ou foi diferente do seu padrão normal?"
- "O que geralmente faz você confiar (ou desconfiar) de algo que vê em rede social?"

**Fechamento (5 min)**
- "Tem alguma coisa sobre como você usa redes sociais que a gente não conversou e que você acha importante eu saber?"

### 4.2 Template de síntese do Journey Map (preencher após cada entrevista)

Para cada participante, organizar os dados em 5 etapas:

| Etapa | Ações | Pensamentos | Emoções | Pontos de dor | Oportunidades |
|---|---|---|---|---|---|
| 1. Gatilho (o que viu, onde) | | | | | |
| 2. Exploração (busca por mais informação) | | | | | |
| 3. Avaliação (comparação, dúvida, validação social) | | | | | |
| 4. Decisão (ação tomada) | | | | | |
| 5. Pós-decisão (arrependimento, satisfação, compartilhamento) | | | | | |

Depois de preenchido para múltiplos participantes de cada grupo, sobrepor os journey maps de Millennials e Alpha lado a lado para identificar divergências por etapa — é aí que H1, H2 e H3 serão testadas com dados reais (ou refutadas).

---

## 5. Plano de análise e síntese

1. **Diary Study**: análise temática das entradas — categorizar motivos de abertura do app, tipos de decisão gerada, padrões de horário/emoção. Comparar frequência de categorias entre os dois grupos.
2. **Journey Mapping**: affinity mapping entre as jornadas coletadas, buscando pontos de convergência (padrão geracional) e divergência (variação individual).
3. **Triangulação**: cruzar os achados qualitativos com os dados de mercado já levantados (TIC Kids Online, We Are Social) para checar se o comportamento observado nos participantes confirma, complementa ou contradiz as tendências macro.
4. **Saída esperada**: 2-3 personas (uma ou mais por geração), um journey map comparativo, e uma lista de "insight statements" priorizados por relevância/frequência — que alimentarão o Projeto 2 (Pesquisa Avaliativa) como hipóteses de design a serem testadas.

---

## 6. Limitações a declarar no relatório final

- O recorte "Geração Alpha" usado (13-17 anos) é uma aproximação operacional; a literatura não tem consenso sobre o corte etário exato dessa geração.
- Amostra qualitativa pequena (Diary Study + Journey Mapping) não é estatisticamente representativa — serve para gerar hipóteses, não para validá-las em escala (isso fica para o Projeto 3, quantitativo).
- Dados de mercado citados como pano de fundo (We Are Social, Opinion Box, TIC Kids Online) refletem tendências nacionais agregadas e não necessariamente o perfil dos participantes recrutados.
